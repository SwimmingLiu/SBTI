# 首页移动端与分享卡收口改版设计

**日期：** 2026-04-16

**目标：**
- 收紧首页移动端布局，让用户在手机上进入首页后，不需要先长距离下滚就能切换并进入任意一个测试。
- 删除 `SDTI` 页面中不需要的 `@Egeria` 标识。
- 统一修复 `SBTI / SDTI / HERTI` 三套分享卡中的文字越界问题，并把二维码角标文案改为更清晰的测试入口提示。

## 当前问题

1. 首页移动端层级过深，外层容器和卡片都带了较多 `padding`，导致内容区偏窄，三套测试入口首屏可见性不足。
2. 首页卡片上存在一部分冗余标签，例如“现有题库”“结果分享”，对进入测试的决策帮助不大，却会占用移动端垂直空间。
3. `SDTI` 页面底部仍保留 `@Egeria` 文案，不符合当前页面需求。
4. 三套分享卡的文字信息区都存在“正文与二维码角标互相抢宽度”的问题；当结果文案较长时，预览图会出现文字顶边、挤压或超出视觉边界的风险。
5. 当前二维码角标文案是“扫码查看入口”，语义不够直接，而且在现有布局里容易换行。

## 设计决策

### 1. 首页移动端改为“测试切换 + 当前卡片”结构

- 桌面端继续保留现有三列卡片主结构，本轮只重点调整 `md` 以下移动端布局。
- 首页移动端顶部保留题库标题区，但整体收紧上下留白，避免首屏被外层装饰性空间吃掉。
- 测试入口区改成三项切换控件：`SBTI / SDTI / HERTI`。
- 切换控件下方只展示“当前选中测试”的精简详情卡，不再在移动端一次性铺开三张完整卡片。
- 当前详情卡仅保留：
  - 测试名
  - 开放状态
  - 一句短描述 / tagline
  - 题量与结果数
  - 主入口按钮
- 删除对进入测试帮助不大的冗余标签，避免移动端卡片继续被碎信息撑高。

### 2. 首页小程序入口仅保留桌面端

- 首页移动端不再显示“小程序入口”区块。
- 非小程序移动环境继续沿用现有自动跳转逻辑；小程序环境下直接展示测试入口，不再额外提示。
- 桌面端保留现有小程序入口区域，继续承接扫码场景。

### 3. `SDTI` 页面移除 `@Egeria`

- 直接删除 `SDTI` 页面底部的 `@Egeria` 文案。
- 不新增替代标识，不保留占位。

### 4. 分享卡统一改为“主内容全宽 + 底部 CTA 横条”

- 不再把二维码角标塞进标题区右上角，也不再通过压缩正文宽度给二维码让位。
- 三套分享卡统一采用同一原则：
  - 标题全宽显示
  - 摘要正文全宽显示
  - 二维码 CTA 放到内容卡底部横条
- 底部 CTA 横条结构为：
  - 左侧二维码
  - 右侧单行文案：`小橙有门 · 扫码开始测试`
- `小橙有门 · 扫码开始测试` 必须保持单行，不允许被换行拆开。

### 5. 分享图使用“适合传播的摘要”，不强塞完整长文

- 结果页正文保持完整，不改页面本身的阅读体验。
- 只有分享图里的文案改成更短、更适合卡片阅读的摘要版本。
- 这样可以减少文字越界风险，也让分享图更像分享卡，而不是整页正文截图。
- 三套测试都采用同一策略，但每套测试保留各自的视觉风格。

## 测试策略

- 单元测试用于覆盖分享图辅助逻辑、文本生成与二维码角标规则。
- e2e 测试用于验证：
  - 首页移动端入口结构
  - `SDTI` 页面不再出现 `@Egeria`
  - 分享弹窗在三套测试里都能正常打开
- 视觉回归重点覆盖：
  - 首页移动端首屏
  - `SBTI / SDTI / HERTI` 分享弹窗预览图
  - `SDTI` 长文案结果场景
- 必跑：
  - `npm test`
  - `npx playwright test`
  - `npx playwright test tests/e2e/visual-regression.spec.ts`

## 预期结果

- 首页移动端首屏优先展示三套测试入口，进入路径更短。
- 首页移动端卡片不再被冗余标签和过大 `padding` 撑高。
- `SDTI` 页面中的 `@Egeria` 被彻底移除。
- 三套分享卡在长文案场景下仍然保持边界稳定，不再出现文字出框或正文被二维码挤成狭窄窄列的问题。
- 二维码角标统一改为更直接的“扫码开始测试”，并在真实生产页面中保持稳定排版。
